Automated Attendance Data Processing System

By Hubert Huang 23 August 2024

Due to some information in this project being confidential, the complete source code will not be disclosed.

Background of the Issue

My internship company (hereinafter referred to as Company A) has outsourced its cleaning and security services to two different companies: Company B for cleaning and Company C for security. Both the cleaning staff and security personnel work different shifts, such as morning, evening, or night, with separate schedules for weekdays and weekends.

Company A’s management department’s partial daily workflow is shown in the diagram below:

Due to the different and complex shifts for cleaning and security personnel, calculating attendance status daily is very time-consuming.

Given the high level of repetition in the aforementioned calculation process and the frequent errors in manual calculations, I will use programming languages to automate it.

Primary Tools Used

This project primarily uses Python as the main programming language, utilizing the Datetime module for time format conversions and calculations, and the Pandas module for handling interactions between Excel, .CSV files, and data operations.

Main Workflow

The main process will be divided into the following steps:

  1. Access raw data
  2. Shift data configuration
  3. Determine the clock-in and clock-out times for each record
  4. Calculate attendance status
  5. Output calculation results

Step2: Shift data configuration

We first define the time format conversion functions from the datetime module to simplify the code for later use.

The interval_transformation function will use the value of controller to determine which time to use as the reference within a flexible time period, and then convert that time string into a time format.

def time_transformation(time_str):
    return datetime.datetime.strptime(time_str, "%H:%M:%S")


def date_transformation(date_str):
    return datetime.datetime.strptime(date_str, "%Y/%m/%d %H:%M:%S")


def str_transformation(time):
    return time.strftime("%Y/%m/%d %H:%M:%S")


def time_transformation2(timedelta):
    return timedelta.strftime("%Y/%m/%d %H:%M:%S")


"""
The value of controller will determine which time to use as the reference for a flexible time period.
If the value is 0, the earlier time is used as the reference; if the value is 1, the later time
is used as the reference.
"""
def interval_transformation(interval_str, controller):  # interval_str format: "hh:mm:ss-hh:mm:ss" or "hh:mm:ss"
    if "-" not in interval_str:
        return date_transformation(today_str + " " + interval_str)
    else:
        return date_transformation(today_str + " " + (interval_str.split("-"))[controller])

Step4: Calculate attendance status for each shift

We define a function to subtract the shift working hours from the employee’s attendance periods to avoid counting the employee’s attendance multiple times.

"""
Define a function to subtract the shift working hours from the employee attendance period.
time_list represents the employee's attendance periods, start is the shift start time, and end is the shift end time.
"""
def clear(time_list, start, end):
    counter = 0
    while (start < date_transformation(time_list[2 * counter]) or
           end > date_transformation(time_list[2 * counter + 1])):
        counter += 1
        if counter > (len(time_list) / 2 - 1):
            break
    if counter <= (len(time_list) / 2 - 1):
        time_list.insert((2 * counter + 1), str_transformation(start))
        time_list.insert((2 * counter + 2), str_transformation(end))
    return time_list

Since employees who assist with substitute shifts sometimes use a card named “Substitute” for clocking in and sometimes use their own cards, we cannot directly determine whether an employee is a substitute based on the card name used for clocking in. Thus, I designed a method to detect whether such irregular attendance data indicates any abnormalities in attendance.

As shown in the image below, the orange line represents the working hours of a shift, the blue line represents employee D’s attendance period, and the green line represents employee E’s attendance period. Assume that employee E is assisting employee D by covering the entire shift’s working hours.

Since employee D’s clock-out time is earlier than the shift end time, we set employee D’s clock-out time as “boundary point X.” Similarly, because employee E’s clock-in time is later than the shift start time, we set employee E’s clock-in time as “boundary point Y” to segment the time periods.

Then we can create segments A, B, and C by pairing the shift start time, shift end time, and each of the boundary points with their adjacent neighbors. At this point, we just need to check whether the number of employees present in each segment A, B and C is greater than 1 to determine if there are employees present throughout the entire shift working hours.

We achieve the creation of boundary points by defining the ordering function.

# Define a function to insert a specific time into a time-sorted list containing different times.
def ordering(inserted_time, time_list):
    counter = 1
    while date_transformation(inserted_time) > date_transformation(time_list[counter]):
        counter += 1
        if counter >= len(time_list):
            break
    time_list.insert(counter, inserted_time)

Next, we apply the ordering function within the new boundary_check function to complete the entire verification process.

"""
Define a function to check shift substitutions and the number of employees present,
and return [number of employees present, substitute01, substitute02...]
start_str = shift start time string
end_str = shift end time string
people_dict = a dictionary that records the clock-in and clock-out times of employees and substitutes.
"""
def boundary_check(start_str, end_str, people_dict):
    shift_start = date_transformation(start_str)
    shift_end = date_transformation(end_str)
    time_line = [shift_start, shift_end]
    result_list = []
    # Use a for loop to apply the "ordering" function to all employees in the dictionary to derive the boundary points.
    for people1 in people_dict.keys():
        for k in range(len(people_dict[people1]) // 2):
            in_time1 = date_transformation(people_dict[people1][2 * k])
            out_time1 = date_trnasformation(people_dict[people1][2 * k + 1])
            if (in_time1 > shift_start and
                in_time1 < shift_end and
                people_dict[people1][2 * k] not in time_line):
                ordering(inserted_time=people_dict[people1][2 * k], time_list=time_line)
            if (out_time1 < shift_end and
                out_time1 > shift_start and
                people_dict[people1][2 * k + 1] not in time_line):
                ordering(inserted_time=people_dict[people1][2 * k + 1], time_list=time_line)
    # Check the number of employees present in each segment between adjacent boundary points.
    for i in range(len(time_line) - 1):
        start = date_transformation(time_line[i])
        end = date_transformation(time_line[i + 1])
        counter = 0  # Record the number of employees present in the current segment.
        replace = []  # A list that stores individuals who successfully substituted using a "Substitute" card.
        for people2 in people_dict.keys():
            for j in range(len(people_dict[people2]) // 2):
                in_time2 = date_transformation(people_dict[people2][2 * j])
                out_time2 = date_transformation(people_dict[people2][2 * k + 1])
                if in_time2 <= start and out_time2 >= end:
                    counter += 1
                    if "Substitue" in people2 and people2 not in replace:               
                        replace.append(people2)
        result_list.append(counter)
    minimum_attend = [float("inf")]
    for result in result_list:
        if result < minimum_attend[0]:
            minimum_attend = [result]
    return minimum_attend + replace

Step5: Output calculation results

In the final part, we need to first create a DataFrame format to facilitate the subsequent generation of the Excel file.

# Create a dictionary for the DataFrame to facilitate the final file output.
result = {today_str:[], "Check_results":[]}

Next, we define the basic_generate function to handle the addition of data for file generation. We then use this function to construct the subsequent generate and result_generate functions, which will be used to produce a complete attendance status for a shift.

# Define a function to add the content to be produced into the DataFrame.
def basic_generate(left_list, right_list, left_arg, right_arg):
    left_list.append(left_arg)
    right_list.append(right_arg)


# Use the previously defined "basic_generate" function to fully generate the attendance status for a shift.
def result_generate(name, required, attend, replacement):
    basic_generate(left_list=result[today_str], right_list=result["Check_results"], left_arg=" ", right_arg=" ")
    def generate(status):
        basic_generate(left_list=result[today_str], right_list=result["Check_results"], left_arg=name, right_arg=status)
        basic_generate(left_list=result[today_str], right_list=result["Check_results"], left_arg="Expected_attendees", right_arg=required)
        basic_generate(left_list=result[today_str], right_list=result["Check_results"],
        left_arg="Actual_attendees", right_arg=attend)
    if required <= attend:
        generate("Attendance_normal")
        if len(replacement) > 0:
            for i in replacement:
                basic_generate(left_list=result[today_str], right_list=result["Check_results"], left_arg="Substitute employee", right_arg=i)
    else:
        generate("Attendance_abnormal")
        if len(replacement) > 0:
            for i in replacement:
                basic_generate(left_list=result[today_str], right_list=result["Check_results"], left_arg="Substitute employee", right_arg=i)

Future Benefits

Firstly, automation resolves the issue of calculation errors that occur during manual operations, mitigating the additional risks and subsequent costs associated with human mistakes.

Secondly, it reduces daily labor costs. According to the management department, the manual calculation process previously required about 30 minutes each day for data verification and comparison, so automation can effectively save labor costs associated with repetitive tasks.
